Method to authenticate the end-user

public authenticate([string $scope = 'openid' ][, string $claims = null ][, string $qoa = null ][, string $qor = null ][, string $locale = null ][, string $loginHint = null ][, string $prompt = null ][, int $maxAge = null ]) : void

After calling this method, make sure to check if any error have occurred

As per the OAuth 2.0 Security Best Current Practice, as PKCE is not used, the state parameter SHOULD be used for CSRF protection and can therefore not be specified as a parameter by the RP; instead, for example, store the client's state in the session, retrieve the state used in the call and reconcile them afterwards.

Parameters
$scope : string = 'openid'

The scope requested. Default value is 'openid'. Valid values are any combination of the following 'openid', 'profile', 'email', 'phone', 'address'

$claims : string = null

The claims requested. Future placeholder. Currently not supported.

$qoa : string = null

The Quality of Authentication (QoA) requested. Valid values are 'qoa1', 'qoa2'

$qor : string = null

The Quality of Registration (QoR) requested. Valid values are 'qor0', 'qor1', 'qor2'

$locale : string = null

The language of the end-user interface. Valid values are 'de', 'fr', 'it', 'en'

$loginHint : string = null

The login hint

$prompt : string = null

Whether and for what the IdP should prompt the end-user. Valid values are 'none', 'login', 'consent'

$maxAge : int = null

The allowable elapsed time in seconds since the last time the end-user was actively authenticated by SwissID. A valid value is an integer >= 0

Method to initiate the step-up of the end-user

public stepUp(string $stepUpType[, string $scope = null ][, string $claims = null ][, string $qoa = null ][, string $qor = null ][, string $locale = null ][, string $loginHint = null ][, string $prompt = null ][, int $maxAge = null ]) : void

By default, most parameters are assumed to be the same as at the time of authentication, but if needed, they can be changed for initiating the step-up.

After calling this method, make sure to check if any error have occurred

As per the OAuth 2.0 Security Best Current Practice, as PKCE is not used, the state parameter SHOULD be used for CSRF protection and can therefore not be specified as a parameter by the RP; instead, for example, store the client's state in the session, retrieve the state used in the call and reconcile them afterwards.

Parameters
$stepUpType : string

The type of step-up to initiate. Valid values are 'LOT1'

$scope : string = null

The scope requested, if different from at the time of authentication. Default value is 'openid'. Valid values are any combination of the following 'openid', 'profile', 'email', 'phone', 'address'

$claims : string = null

The claims requested, if different from at the time of authentication. Future placeholder. Currently not supported.

$qoa : string = null

The Quality of Authentication (QoA) requested, if different from at the time of authentication. Valid values are 'qoa1', 'qoa2'

$qor : string = null

The Quality of Registration (QoR) requested, if different from at the time of authentication. Valid values are 'qor0', 'qor1', 'qor2'

$locale : string = null

The language of the end-user interface, if different from at the time of authentication. Valid values are 'de', 'fr', 'it', 'en'

$loginHint : string = null

The login hint, if different from at the time of authentication.

$prompt : string = null

Whether and for what the IdP should prompt the end-user, if different from at the time of authentication. Valid values are 'none', 'login', 'consent'

$maxAge : int = null

The allowable elapsed time in seconds since the last time the end-user was actively authenticated by SwissID, if different from at the time of authentication. A valid value is an integer >= 0
